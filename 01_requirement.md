# 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 목적

광고 업체가 수행한 작업 결과를 데이터로 기록하고, 광고주에게 성과를 보여주는 서비스. 네이버 검색 결과에서 특정 콘텐츠(플레이스, 블로그, 카페)의 노출 순위를 확인하고 일별 순위 변동을 트래킹한다.

### 1.2 사용자 유형

| 유형 | 설명 | 주요 역할 |
|------|------|-----------|
| 관리자 | 시스템 관리자 (회원가입 없음, 초기 데이터) | 회원 승인, 업체-광고주 매핑 관리 |
| 업체 | 광고 대행사 | 순위 추적 등록, 작업 내역 기록, 데이터 관리 |
| 광고주 | 광고를 의뢰한 고객사 | 본인에게 매핑된 데이터 조회 (열람만 가능) |

### 1.3 사용자 관계

- 업체와 광고주는 N:N 관계
- 업체가 데이터 등록 시 해당 데이터를 볼 광고주를 지정
- 광고주는 본인에게 매핑된 데이터만 조회 가능

## 2. 기능 요구사항

### 2.1 플레이스 순위

네이버 검색 시 플레이스 영역 내 순위를 조회/추적

#### 2.1.1 실시간 플레이스 순위 (건바이건)

| 항목 | 내용 |
|------|------|
| 입력 | 키워드, URL |
| 출력 | 현재 순위 |
| 저장 | DB 저장 안함 |

#### 2.1.2 플레이스 순위 추적 (월보장)

| 기능 | 설명 |
|------|------|
| 등록 | 키워드, URL, 광고주 지정 |
| 목록 조회 | 등록된 추적 목록 |
| 상세 조회 | 일별 순위 히스토리 |
| 수정 | 추적 정보 수정 |
| 상태 변경 | 추적 중단 (삭제 없음, 데이터 보존) |

**회차 개념:**
- 순위가 확인된(노출된) 횟수 25회 기준으로 회차 구분
- 순위권 외(미노출)인 날은 카운트 안됨
- 예: 1/1 3위(1번) → 1/2 5위(2번) → 1/3 순위권 외(카운트 안됨) → 1/4 2위(3번) → ... → 25회 노출 달성 시 2회차 시작
- 계약 종료 전까지 순위 추적 지속 (계약 여부와 무관하게 추적)
- 관리자가 계약 상태에 따라 추적 중단 결정

**자동 크롤링:**
- 매일 특정 시간(예: 오후 1시)에 자동 순위 조회
- 조회 결과 DB 저장
