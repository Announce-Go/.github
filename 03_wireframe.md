# 화면 설계서

## 1. 화면 목록

### 1.1 공통 화면

| 화면명 | 경로 | 설명 |

## 2. 공통 화면 상세

### 2.1 로그인 화면
- **경로**: `/login`

#### 레이아웃
```
┌─────────────────────────────────────┐
│         알리다고 로고/제목           │
│                                     │
│  ┌───────────────────────────┐     │
│  │  아이디                   │     │
│  └───────────────────────────┘     │
│  ┌───────────────────────────┐     │
│  │  비밀번호                 │     │
│  └───────────────────────────┘     │
│                                     │
│  □ 로그인 상태 유지                 │
│                                     │
│      [로그인]                       │
│                                     │
│  ─────────────────────────────      │
│                                     │
│        회원가입                     │
└─────────────────────────────────────┘
```

#### 데이터 처리
- 로그인 API 호출
  - 요청: 로그인ID, 비밀번호, 로그인상태유지여부
  - 응답: 인증토큰, 사용자타입(admin/agency/hospital), 승인상태

#### 화면 전환
- 로그인 성공:
  - 관리자 → `/admin/dashboard`
  - 광고업체 → `/agency/dashboard`
  - 병원 → `/hospital/dashboard`
- 승인 대기: "승인 대기 중" 메시지 표시
- 회원가입 링크 → `/register`

---

- 가입 완료 → 로그인 화면으로 이동 (승인 대기 안내)
- 취소 → 로그인 화면

#### 특이사항
- 사용자 종류 선택에 따라 입력 폼 동적 변경
- 최소 1개 이상의 마케팅 타입 선택 필수

- 승인대기 카드 클릭 → `/admin/approvals`
