### 질문 내용

블로그 상위 노출에서 노출 형태가 "건바이건"일 때의 업무 프로세스와 데이터 입력 방식 확인이 필요합니다.

### 확인 필요 사항

1. **건바이건의 업무적 의미**:
   - 건바이건이 업무적으로 어떤 계약/운영 방식을 의미하나요?
   - 월보장과 어떤 차이가 있나요?

2. **회차 방식과 동일한 UI 사용**:
   - "회차 방식과 같은 UI에서 처리한다"는 것이 무엇을 의미하나요?
   - 건바이건도 25일 단위로 관리되나요?
   - 건바이건의 데이터 입력 방식은 어떻게 되나요?

## 블로그 상위 노출 - 회차 전환 방식

- **등록일**: 2025-11-19
- **관련 섹션**: [3.2.2 마케팅 타입별 데이터 입력](requirement.md#322-마케팅-타입별-데이터-입력)

### 질문 내용

블로그 상위 노출에서 25일 데이터 입력 완료 후 다음 회차로 전환되는 방식을 확인해야 합니다.

### 확인 필요 사항

1. **회차 전환 방식**:
   - 25일 입력 완료 시 자동으로 다음 회차가 생성되나요?
   - 관리자가 확인 후 수동으로 다음 회차를 생성하나요?
   - 업체가 직접 다음 회차를 생성할 수 있나요?

2. **관리자 승인이 필요한 경우**:
   - 승인 전까지 업체는 데이터 입력을 못하나요?
   - 승인 대기 상태를 표시해야 하나요?
   - 승인 시 알림이 필요한가요?

3. **회차 간 연속성**:
   - 이전 회차와 다음 회차 사이에 공백 기간이 있어도 되나요?
   - 키워드가 변경될 수 있나요?

## 복수 업체의 동일 마케팅 타입 처리

- **등록일**: 2025-11-19
- **관련 섹션**: [2.3 업체-병원 관계](requirement.md#23-업체-병원-관계)

### 질문 내용

하나의 병원에 여러 광고업체가 담당할 수 있는 N:1 구조에서 동일한 마케팅 타입을 여러 업체가 담당하는 경우의 처리 방식 확인이 필요합니다.

### 확인 필요 사항

1. **중복 마케팅 타입 가능 여부**:
   - 하나의 병원에 2개의 업체가 있을 때, 두 업체 모두 "블로그 상위 노출"을 선택할 수 있나요?
   - 다른 마케팅 타입(브랜드 블로그, 카페, 영수증 블로그)도 중복 가능한가요?

2. **업체별 회차 관리**:
   - 두 업체가 모두 블로그 상위 노출을 담당한다면, 회차는 업체별로 독립적으로 관리되나요?
   - 아니면 병원 단위로 통합 관리되나요?

3. **데이터 표시 방식**:
   - 관리자가 조회할 때: 업체별로 구분해서 표시하나요? 통합해서 표시하나요?
   - 병원이 조회할 때: 여러 업체의 동일 마케팅 타입 데이터를 어떻게 구분해서 보여주나요?
   - 키워드가 겹치는 경우 어떻게 처리하나요?

## 키워드 관리 방식

- **등록일**: 2025-11-19
- **관련 섹션**: [3.2.2 마케팅 타입별 데이터 입력](requirement.md#322-마케팅-타입별-데이터-입력)

### 질문 내용

블로그 상위 노출과 브랜드 블로그에서 사용하는 키워드의 관리 방식 확인이 필요합니다.

### 확인 필요 사항

1. **키워드 입력 방식**:
   - 자유 텍스트로 입력하나요?
   - 미리 정의된 키워드 목록에서 선택하나요?
   - 자동완성 기능이 필요한가요?

2. **키워드 수 제한**:
   - 블로그 상위 노출: 회차당 키워드 수 제한이 있나요?
   - 브랜드 블로그: 키워드 수 제한이 있나요?

3. **키워드 일관성**:
   - 같은 병원에 대해 업체들이 사용하는 키워드를 통일해야 하나요?
   - 키워드 표준화나 정규화가 필요한가요?
