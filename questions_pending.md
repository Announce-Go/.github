## 광고업체 입력 데이터 유형 확인

- **등록일**: 2025-11-18
- **관련 섹션**: [3.2.1 광고 데이터 입력](requirement.md#321-광고-데이터-입력)

### 질문 내용

초기 요구사항 청취 시 광고업체가 입력해야 하는 데이터가 **7종류**라고 들었으나, 현재 정의된 항목은 **2가지**(광고 순위 데이터, 계약 체결 정보)만 있습니다.

나머지 **5가지 데이터 유형**이 무엇인지 확인이 필요합니다.

### 추정 항목 (확인 필요)

병원 통계 화면에 표시될 것으로 추정되는 항목들:
- 네이버 광고
- 파워 컨텐츠
- 블로그 상위 노출
- 기타 항목들?

### 확인 필요 사항

1. **7가지 데이터 유형 목록**: 광고업체가 입력해야 하는 전체 데이터 유형 목록
2. **각 유형별 입력 항목**: 각 데이터 유형마다 입력해야 하는 구체적인 필드
   - 예) 네이버 광고: 키워드, 순위, 날짜 등
   - 예) 파워 컨텐츠: ?
   - 예) 블로그 상위 노출: ?
3. **데이터 구분 필요성**: 각 유형을 별도로 관리해야 하는지, 아니면 통합하여 "광고 순위 데이터"로 관리할 수 있는지?
4. **병원 화면 표시 방식**: 각 유형별 데이터가 병원 화면에 어떻게 구분되어 표시되는지?

### 추가 질문

- 현재 카카오톡으로 받고 있는 데이터 형식의 예시를 제공해주실 수 있나요?
- 광고업체마다 입력하는 데이터 유형이 다른가요? (예: A업체는 네이버 광고만, B업체는 블로그만)

## 병원 통계 화면 데이터 연관성 확인

- **등록일**: 2025-11-18
- **관련 섹션**: [3.3.1 광고 현황 조회](requirement.md#331-광고-현황-조회-조회-전용)

### 질문 내용

병원 통계 화면 예시 캡처본에 표시되는 항목들이 광고업체가 입력하는 데이터와 어떻게 연관되는지 명확한 설명이 필요합니다.

### 확인 필요 항목

캡처본에서 보이는 다음 항목들의 데이터 출처 및 계산 방식:

1. **예산**:
   - 광고업체가 직접 입력하는 데이터인가요?
   - 계약 체결 정보의 "결제금액"과 동일한가요?
   - 아니면 별도로 입력해야 하는 항목인가요?

2. **남은 금액**:
   - 자동 계산되는 값인가요? (예산 - 사용금액)
   - 광고업체가 입력하는 데이터인가요?
   - 계산 공식이 있다면 무엇인가요?

3. **견적**:
   - 이것은 어떤 데이터를 의미하나요?
   - 광고업체가 입력하는 항목인가요?
   - 예산과 어떻게 다른가요?

4. **기타 표시 항목**:
   - 캡처본에 있는 다른 항목들도 모두 설명 필요
   - 각 항목이 입력 데이터인지, 계산 값인지 구분

### 요청 사항

- **캡처본 기반 설명**: 실제 캡처본의 각 영역에 대해 "이 부분은 광고업체가 입력한 XXX 데이터입니다" 또는 "이 부분은 YYY와 ZZZ를 계산한 값입니다" 형식으로 상세 설명
- **입력 항목 정리**: 광고업체가 실제로 입력해야 하는 전체 항목 목록
- **계산 항목 정리**: 시스템에서 자동으로 계산/표시되는 항목 목록과 계산 공식

## 병원 통계 데이터 이력 관리 및 명칭 확인

- **등록일**: 2025-11-18
- **관련 섹션**: [3.3.1 광고 현황 조회](requirement.md#331-광고-현황-조회-조회-전용)

### 질문 내용

병원 통계 화면에 년월일이 표시되는데, 데이터의 생성 주기와 이력 조회 가능 여부, 그리고 이 데이터를 내부적으로 어떻게 부를지 확인이 필요합니다.

### 확인 필요 사항

1. **데이터 생성 주기**:
   - 하루에 한 번씩 생성되는 정보인가요?
   - 실시간으로 업데이트되는 정보인가요?
   - 광고업체가 데이터를 입력할 때마다 갱신되는 것인가요?

2. **과거 데이터 조회**:
   - 병원이 과거 날짜의 통계 정보를 볼 수 있어야 하나요?
   - 예시: 2025.11.17에 접속했을 때, 2025.11.10의 통계 정보도 조회 가능해야 하는지?
   - 조회 가능하다면 얼마나 오래된 데이터까지 조회 가능한가요? (예: 최근 1개월, 3개월, 1년 등)

3. **데이터 이력 관리**:
   - 날짜별로 데이터를 히스토리로 저장해야 하나요?
   - 아니면 최신 데이터만 유지하면 되나요?

4. **데이터 명칭**:
   - 이 통계 정보를 시스템 내부적으로 뭐라고 부르면 좋을까요?
   - 예시: 리포트, 통계, 일일 리포트, 광고 성과 리포트, 대시보드 데이터 등

### 추가 질문

- **데이터 비교 기능**: 날짜별 데이터를 비교하는 기능이 필요한가요? (예: 11.10 vs 11.17 비교)
- **기간별 조회**: 특정 기간의 데이터를 조회하는 기능이 필요한가요? (예: 2025.11.01 ~ 2025.11.17)
- **데이터 수정**: 과거 날짜의 데이터를 광고업체가 수정할 수 있어야 하나요?
