## "계약 체결"의 정확한 의미 및 프로세스

- **등록일**: 2025-11-13
- **답변일**: 2025-11-14
- **관련 섹션**: [3.2.3 계약 체결 관리](requirement.md#323-계약-체결-관리), [3.1.4 계약 체결 관리](requirement.md#314-계약-체결-관리)

### 질문 내용

"계약 체결"의 정확한 의미를 명확히 해주세요:
- 알리다고와 광고업체 간의 계약?
- 광고업체와 병원 간의 계약?
- 결제 연동 기능이 필요한가요? (PG사 연동)

### 결정 사항

✅ **결제 연동 기능 없음** (PG 연동 없이 수동 관리)

**프로세스**:
1. 알리다고와 광고업체 간 계약 체결 후 시스템 사용 시작
2. **광고업체 입력 항목**: 은행명, 계좌번호, 입금주, 결제금액, 결제일, 계약만료일
3. **알리다고 확인 작업**: 실제 입금 확인 후 상태를 "결제완료"로 변경
4. **역할 분담**: 입력은 광고업체, 최종 확인 및 상태 변경은 알리다고

## 광고업체 계정 생성 시 비밀번호 처리 방법

- **등록일**: 2025-11-13
- **답변일**: 2025-11-14
- **관련 섹션**: [3.1.2 광고업체 계정 관리](requirement.md#312-광고업체-계정-관리)

### 질문 내용

광고업체 계정을 관리자가 생성할 때 비밀번호는 어떻게 처리하나요?
- 관리자가 직접 설정하여 전달?
- 시스템이 자동 생성 후 이메일 발송?
- 초기 비밀번호 설정 후 최초 로그인 시 변경 강제?

### 결정 사항

✅ **관리자가 계정정보(ID/초기 비밀번호)를 부여**하고, 광고업체는 **최초 로그인 시 비밀번호 변경창에서 비밀번호를 변경해야 대시보드로 이동 가능**
