# 광고 관리 시스템 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
- 현재 120여개 이상의 병원 광고를 카카오톡 단톡방으로 관리하고 있으나, 관리의 복잡성으로 인한 효율성 저하
- 여러 광고 업체의 서비스를 한 곳에서 통합 관리할 수 있는 시스템 필요
- 병원이 자신의 광고 상황(순위, 노출 등)을 실시간으로 확인할 수 있는 플랫폼 제공

### 1.2 비즈니스 배경
- 한의원 광고 관리의 차별화 포인트로 활용
- 투명한 광고 성과 공유를 통한 신뢰 구축

---

## 2. 시스템 구성

### 2.1 기본 정보
- **서비스명**: 알리다고(AnnounceGo) 어드민 사이트(가칭)
- **플랫폼**: PC 웹사이트 중심 (모바일은 추후 개발)
- **공개 범위**: 비공개 시스템 (검색엔진 노출 차단)
- **예상 데이터량**: 하루 최대 600건 이상 (120개 업체 × 약 5건)

### 2.2 사용자 역할

#### 2.2.1 관리자 (알리다고)
- 모든 광고업체 및 병원 정보 조회/관리
- 광고업체 계정 생성 및 관리
- 계약 결제 상태 관리 (미결제/결제완료 수정 권한)
- 모든 1:1 문의 내역 확인 가능

#### 2.2.2 광고업체 (120개)
- 담당 병원의 광고 순위 데이터 입력
- 자사의 계약 결제 정보 조회
- 광고 계약 만료일 D-day 확인
- 관리자와 1:1 문의/소통

#### 2.2.3 병원 (120개)
- 로그인 없이 사용하는 사용자
- 자신의 광고 상황(순위, 노출 등) **조회 전용**

### 2.3 업체-병원 관계
- **1:1 매칭 구조**: 각 광고업체는 1개의 병원을 담당

## 3. 주요 기능 요구사항 (역할별)

### 3.1 관리자 기능

#### 3.1.1 광고업체 계정 관리
- 광고업체 계정 생성 및 고유 ID 부여
- 필수 정보 항목 (변경 가능):
    - 업체명
    - 담당 병원명
    - 연락처
    - 이메일
    - 기타 관리 정보

#### 3.1.2 전체 데이터 조회
- 모든 광고업체의 순위 데이터 조회
- 모든 병원의 광고 현황 조회
- 통계 및 리포트 기능

#### 3.1.3 계약 결제 관리
- 모든 광고업체의 계약 결제 정보 조회
- 계약 결제 상태 변경 (미결제 ↔ 결제완료)
- 계약 결제 이력 관리

#### 3.1.4 문의 관리
- 모든 광고업체의 1:1 문의 내역 확인
- 문의에 대한 답변 작성

### 3.2 광고업체 기능

#### 3.2.1 광고 데이터 입력
담당 병원의 광고 순위 데이터 입력:
- 키워드
- 순위
- 날짜
- 기타 광고 성과 지표

#### 3.2.2 데이터 조회
- 담당 병원의 입력한 데이터 조회
- 데이터 입력 내역 히스토리

#### 3.2.3 계약 결제 정보 관리
자사의 계약 결제 정보 입력 및 조회:
- 은행명
- 계좌번호
- 입금주
- 결제금액
- 결제일
- 계약만료일

#### 3.2.4 계약 만료일 알림
- 대시보드에 **계약만료일 D-day 표시**
- 만료 임박 시 시각적 알림 (예: D-7, D-3, D-day)

#### 3.2.5 1:1 문의
- 관리자에게 문의 작성
- 본인 작성 문의 내역 조회
- 관리자 답변 확인

### 3.3 병원 기능

#### 3.3.1 광고 현황 조회 (조회 전용)
- 자신의 광고 순위 확인
- 키워드별 노출 현황 확인
- 광고 성과 그래프/차트

## 6. 요구사항 확인 필요사항 (질문)

**Q1. 광고업체 계정 생성 시 비밀번호 처리 방법**
- 관리자가 광고업체 계정을 생성할 때 비밀번호를 어떻게 설정하나요?
- 옵션1: 관리자가 직접 초기 비밀번호 설정
- 옵션2: 프로그램 자동생성
- 옵션3: 또 다른 방법

**광고업체 비밀번호 변경**: 광고업체가 로그인 후 본인의 비밀번호를 변경할 수 있는 기능이 필요한가요?

**Q2. 주요 기능사항에 나온 각 입력 정보 항목 확인**

**광고업체 계정 정보** (관리자가 입력):
- [ ] 업체명
- [ ] 담당 병원명
- [ ] 연락처
- [ ] 이메일
- [ ] 기타: ___________________

**광고 순위 데이터** (광고업체가 입력):
- [ ] 키워드 (예: "강남 한의원")
- [ ] 순위 (예: 3위)
- [ ] 날짜 (예: 2025-11-13)
- [ ] 플랫폼 (예: 네이버/구글?)
- [ ] 기타: ___________________

**계약 결제 정보** (광고업체가 입력):
- [ ] 은행명
- [ ] 계좌번호
- [ ] 입금주
- [ ] 결제금액
- [ ] 결제일
- [ ] 계약만료일
- [ ] 기타: ___________________

**예시 데이터 필요**: 실제 입력될 데이터의 샘플을 제공해주세요.

**병원이 보게 되는 정보 예시**: 병원이 실제로 확인하게 될 화면에는 어떤 정보들이 어떤 형식으로 표시되는지 예시를 제공해주세요.

**Q3. 병원의 광고 현황 조회 방식**
- 병원이 자신의 광고 상황을 조회하게 되는데, 링크만 있으면 누구나 볼 수 있는 문제가 있습니다.
- 옵션1: 병원도 로그인 필요 (ID/PW 발급)
- 옵션2: 링크 + 간단한 인증 (예: 병원 전화번호 뒷자리)
- 옵션3: 링크 공개 허용
- 옵션4: 기타 방법

**Q4. "계약 결제"의 정확한 의미**

현재 이해한 내용:
- 광고업체가 은행명, 계좌번호, 입금주 등을 입력하는데, 이는 보통 **돈을 받는 사람**이 제공하는 정보입니다.
- 하지만 "계약 결제 상태(미결제/결제완료)"는 관리자가 변경한다고 하여 혼란이 있습니다.

**확인 필요사항**:
1. "계약 결제"는 광고업체와 알리다고 간의 계약이 맞나요?
2. 어떤 계약인가요? (예: 광고 대행 수수료? 시스템 이용료?)
3. 결제 프로세스는 어떻게 되나요?
    - 광고업체가 알리다고에게 돈을 입금?
    - 알리다고가 광고업체에게 돈을 입금?
4. 은행명, 계좌번호를 누가 입력하나요?
    - 돈을 받을 사람의 계좌 정보?
    - 돈을 보낼 사람의 계좌 정보?

**어떤 프로세스인지 설명 부탁드립니다**:

**Q5. 1:1 문의 기능의 UI/UX 형식**
- 현재는 "게시판" 형식으로 작성되어 있으나, 채팅 형식(카카오톡 대화 목록, 대화창)으로 변경 가능한가요?
- 옵션1: 일반적인 게시판 (제목, 내용, 답변)
- 옵션2: 채팅 형식 (실시간 아님, 카톡처럼 대화 흐름)

## 7. 용어 정의

| 용어 | 설명 |
|------|------|
| 관리자 | 알리다고(AnnounceGo) 시스템 운영 주체 |
| 광고업체 | 병원 광고를 담당하는 외부 업체 (120개) |

| D-day | 계약만료일까지 남은 일수 |
